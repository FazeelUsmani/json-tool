# UI primitives — shadcn/ui (generated)

Files in this directory are generated by `npx shadcn add <component>`
and are kept as a vendor layer. Treat them as third-party code that
happens to live in the repo:

- Edit only if upstream has the same change. Otherwise re-run
  `npx shadcn add <component>` to overwrite cleanly. Local edits get
  clobbered on re-add.
- shadcn co-locates the component + its `cva` variant constants in
  one file. `eslint.config.js` disables
  `react-refresh/only-export-components` for this directory because
  of that pattern.
- Active shadcn config: `components.json` at the repo root.

Application code that wraps or extends these primitives lives one
level up in `src/components/` (e.g. tree, table, editor, debug
components import from `@/components/ui/*`).
